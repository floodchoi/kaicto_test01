# Meeting Minutes Summarizer — 기능 정리

회의 녹음·스크립트를 AI로 전사·요약해 팀이 함께 관리하는 웹앱.
(2026-07-20 기준 / 스택: React(Vite)+Tailwind · Vercel 서버리스 · Neon PostgreSQL · Gemini API / 로컬 LLM)

## 1. 계정과 보안

- **이메일/비밀번호 회원가입·로그인** — 비밀번호는 scrypt 해시, 로그인 시 30일 무상태 토큰(HMAC 서명) 발급
- **초대 코드 가입** — 관리자가 코드 여러 개를 만들고 코드별 최대 사용 횟수 지정. 유효한 코드로 가입하면 즉시 사용 가능
- **관리자 승인제** — 코드 없이 가입하면 관리자 승인 후 로그인 가능
- **봇 가입 방지** — 허니팟 필드 + 서버 서명 챌린지(3초~10분) + IP당 요청 제한
- **회의록 원문 암호화 저장** — 원문(raw_text)은 AES-256-GCM으로 암호화되어 DB에 저장. DB를 직접 열람하거나 유출돼도 내용을 읽을 수 없고, 관리자도 타인의 비공개 회의록을 볼 수 없음
- **데이터 격리** — 모든 회의록은 소유자 계정에 묶이며, 본인 것 또는 "전체 공개"로 지정된 것만 열람 가능
- 로그인 타이밍 공격 방지, SQL 인젝션 차단(파라미터 바인딩), XSS 기본 차단, 보안 응답 헤더

## 2. 회의록 작성

- **텍스트 직접 입력** — 회의 스크립트를 붙여넣어 작성
- **초안 자동 임시 저장** — 작성 중 내용이 브라우저에 자동 저장되어, 페이지를 닫거나 새로고침해도 "새 회의록"에서 이어서 작성. 목록 화면에 "✍️ 작성 중인 회의록 — 이어서 작성" 안내 표시
- **요약 없이 저장** — LLM 연결 없이 원문 그대로 저장 (요약·액션 아이템은 나중에 수정에서 추가 가능)

## 3. 회의 녹음 (앱 내)

- **마이크 녹음 / 탭 오디오 + 마이크 녹음** (다른 탭의 화상회의 소리까지 함께 녹음)
- **일시정지·재개**, 화면을 이동해도 녹음 지속 — 헤더에 전역 REC 배지(경과 시간 + 실시간 레벨 미터)
- **입력 레벨 모니터링** — 10초 이상 무음이면 경고 표시
- **5초마다 IndexedDB 임시 저장** — 탭 크래시·정전에도 유실 없음, 재접속 시 자동 복구
- **대체/이어붙임** — 이전 녹음을 지우고 새로 녹음하거나, 뒤에 이어서 녹음
- **보관된 녹음 목록** — 전사에 사용, 파일로 저장(다운로드), 삭제. "변환 후 원본 삭제" 옵션
- 녹음 원본은 서버에 올라가지 않음 (브라우저 로컬 보관, 전사 시에만 Gemini로 직접 전송)

## 4. 오디오 → 텍스트 전사

- **Gemini 멀티모달 전사** — aac/m4a/mp3/wav/webm, 최대 500MB(긴 파일은 자동 분할이라 원본 크기 무관). 브라우저→Gemini 직접 통신(우리 서버 미경유)
- **긴 파일 분할 전사** — 15분 초과 시 10분 조각(16kHz 모노 WAV)으로 나눠 조각마다 결과가 실시간 도착
- **실시간 진행창** — 전사되는 텍스트가 진행 패널에 즉시 표시, 목록 화면으로 이동해도 백그라운드 계속
- **⏹ 전사 중지** — 진행 중 언제든 중지, 완료된 조각까지는 입력란에 보존, 즉시 재시작 가능
- **반복 생성 방어** — 같은 구절 반복 루프 실시간 감지 → 자동 재시도 → 부분 구제, 조각 경계 중복 자동 제거, 반복 잔여물 압축
- **안정성** — 모든 단계(업로드·처리 대기·수신) 타임아웃, 스트림 120초 정지 감시, 네트워크 끊김 시 복구 대기 후 자동 재시도, 오프라인 시 연결 복구까지 대기
- **영어 혼용 지원** — 영어 발화는 EN 원문 + KO 번역 병기
- **제목 자동 추천** — 전사 완료 시 회의 제목 자동 제안 (비어있을 때만)

## 5. AI 요약

- **제공자 선택** — Gemini(클라우드) 또는 로컬 LLM(Ollama·LM Studio 등 OpenAI 호환)
- **출력** — 3줄 요약 · 주요 아젠다(주제/논의) · 액션 아이템(할 일/담당자/기한) · 태그
- **항상 한국어 요약** — 영문 회의도 한국어로 요약(영어로 나오면 자동 감지 후 1회 재요청)
- **긴 회의 대응** — 전사문 전체(중·후반 포함) 반영을 강제하는 프롬프트, 빈 아젠다·액션아이템 방지
- **실시간 진행 로그** — 요청·수신 글자 수·경과 시간을 1~2줄 로그로 표시 (로컬 reasoning 모델의 "사고 중"도 표시)
- **미리보기에서 항목별 편집** — 저장 전 요약 문장·아젠다·액션 아이템·태그를 바로 수정·삭제·추가
- **견고한 파싱** — 코드펜스·<think> 블록·앞뒤 잡담 제거, 중간에 잘린 JSON 복구, 스트리밍/비스트리밍 응답 형식 모두 지원

## 6. 회의록 관리

- **목록** — 카드 뷰 / 테이블 뷰 전환(선택 기억). 테이블은 제목·날짜·프로젝트·태그(2개+N)·공개 컬럼
- **검색·필터** — 키워드(제목·요약·태그) + 날짜 범위 + 프로젝트별
- **상세 보기** — 3줄 요약, 아젠다, 액션 아이템 체크(완료 토글), 원문 보기
- **원문 내 검색** — 원문 보기에서 키워드 하이라이트, Enter/▲▼로 이전·다음 이동
- **수정** — 제목·원문·요약·아젠다·태그·공개 범위·프로젝트·액션 아이템(추가/수정/삭제/완료) 전부 편집, 최종 수정 일시·계정 표시
- **삭제** — 소유자 전용, 확인 후 액션 아이템까지 함께 삭제
- **액션 아이템 탭** — 전체 회의의 액션 아이템 모아보기 + 검색·필터, 완료 체크
- **프로젝트** — 개인 프로젝트 + 관리자가 만드는 공유 프로젝트로 회의록 분류, 모든 화면에서 프로젝트 필터
- **프로젝트 멤버 지정** — 이메일 검색으로 회원을 프로젝트 멤버로 지정. 멤버는 그 프로젝트의 모든 회의록을 열람·수정 가능(비공개 포함, 삭제는 소유자만). 멤버 해제 시 즉시 접근 차단
- **공개 범위** — 나만 보기 / 전체 공개(가입자 열람 가능, 편집·삭제는 소유자만)

## 7. API 키·모델 관리

- **계정별 키 저장** — Gemini 키는 AES-256-GCM 암호화로 내 계정에 저장, 다른 기기에서도 로그인하면 사용
- **무료 → 유료 자동 전환** — 무료 키와 유료(예비) 키를 함께 등록하면, 무료 한도 소진(429) 시 유료 키로 자동 전환해 작업이 끊기지 않음 (전환 시 화면 안내)
- **관리자 키 공유** — 관리자가 허용한 회원은 본인 키 없이 관리자 키 사용. 이때 모델은 관리자가 지정한 모델로 고정
- **전사 전용 키/모델** — 요약과 별도로 전사에 다른 키·모델 지정 가능
- **실시간 모델 목록** — 키로 사용 가능한 Gemini 모델을 실시간 조회해 선택, 로컬 서버의 모델 목록도 조회
- **로컬 LLM 연결** — 서버 주소(/v1 자동 보정)·모델명 설정, CORS 안내

## 7-1. 외부 연동 (Notion · Dooray)

- **Notion 자동 저장** — ⚙️ 설정에 Notion 통합 토큰과 대상(페이지 또는 데이터베이스 URL/ID)을 등록하면, 회의록 저장 시 제목·요약·아젠다·액션 아이템(체크박스)·태그·원문이 자동 기록됨. 데이터베이스는 제목/태그(multi_select)/날짜(date) 속성을 자동 매핑. 저장 후 "Notion에서 열기" 링크 제공
- **Dooray 업무 등록** — Dooray API 토큰과 프로젝트 ID를 등록하면, 저장 시 액션 아이템이 해당 프로젝트의 업무로 자동 등록됨 (아이템당 1건, 최대 20건)
- 토큰은 계정별 AES-256-GCM 암호화 저장, 호출은 서버 경유(브라우저 노출 없음). 설정에서 🔌 연결 테스트 제공, 전송 성공/실패는 활동 로그에 기록되고 저장 직후 배너로 알림
- **↗ 연동 전송(재시도)** — 저장 시 전송이 실패했거나 연동 설정 전에 만든 회의록도, 상세 화면의 "↗ 연동 전송" 버튼으로 나중에 다시 Notion/Dooray로 보낼 수 있음
- **회의록별 전송 선택** — 새 회의록 저장 옵션에서 "📝 Notion에 저장" / "📋 Dooray 업무 등록"을 회의록마다 켜고 끌 수 있음(마지막 선택 기억). 연동 전송이 실패해도 회의록 저장 자체는 항상 성공

## 8. 관리자 기능

- **회원 관리** — 가입 승인/거부, 관리자 키 사용 권한 지정, 회원 삭제
- **초대 코드 관리** — 코드 생성(자동 생성)·삭제, 코드별 최대 사용 횟수, 사용 현황
- **공유 프로젝트** — 전체 회원이 쓰는 프로젝트 생성
- **공유 모델 지정** — 관리자 키 사용 회원이 쓸 요약·전사 모델 강제
- **🔧 원클릭 DB 마이그레이션** — 스키마 오류 배너에서 버튼 한 번으로 migrate.sql 실행(멱등)
- **활동 로그** — 로그인·가입·회의록 생성/수정/삭제·멤버 변경·키 저장·마이그레이션이 자동 기록되고, 브라우저에서 발생한 전사/요약 오류도 서버로 보고됨. 회원 관리 화면에서 최근 100건을 이메일 필터로 조회

## 9. 사용성

- **모바일 대응** — 헤더 아이콘화, 필터 재배치, 카드 1열, 테이블 가로 스크롤
- **도움말 팝업** — 기본 기능 / 데이터 저장 위치 / 보안(암호화) / LLM 연결 방법 안내
- **오류 안내** — 단계별 원인이 담긴 한국어 오류 메시지(네트워크·키·한도·모델·마이그레이션 등)

## 10. 데이터 저장 위치 요약

| 데이터 | 위치 | 비고 |
|---|---|---|
| 회의록 텍스트·요약·액션아이템 | 서버 DB (Neon) | 원문은 AES-256-GCM 암호화 |
| 녹음 원본(오디오) | 브라우저 IndexedDB | 서버 미저장, 전사 시 Gemini로만 전송 |

## 11. 기술 구조 (요약)

- **프론트엔드**: React(Vite) 단일 페이지, Tailwind v4 — AI 호출(전사·요약·모델 목록·업로드)은 전부 브라우저에서 제공자(Gemini/로컬)로 직접
- **백엔드**: Vercel 서버리스 함수 — `/api/auth`(가입·로그인), `/api/me`(내 정보·키), `/api/meetings`(+`/:id`), `/api/action-items`, `/api/projects`, `/api/admin-users`, `/api/invite-codes`, `/api/migrate`
- **DB**: PostgreSQL(Neon) — users / invite_codes / projects / meetings / action_items
- **배포**: GitHub push → Vercel 자동 배포. 환경변수는 `DATABASE_URL`, `AUTH_SECRET` 두 개뿐
- 로컬 개발·배포 절차와 보안 상세는 [README.md](README.md) 참고
